## 1. 정의

현재 화면을 차단하고 사용자의 명시적 결정을 요구하는 다이얼로그.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `overlay` | ✓ | 배경 오버레이 |
| `dialog` | ✓ | 다이얼로그 본체 |
| `title` | ✓ | 모달 제목 |
| `body` | ✓ | 본문 메시지 |
| `actions` | ✓ | 액션 버튼 (취소·확인) |
| `close` | – | X 닫기 버튼 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| size | `sm` \| `md` \| `lg` |
| tone | `default` \| `danger` |

## 4. 토큰 매핑

| 속성 | 토큰 | 값 |
|---|---|---|
| dialog.background | `{alias.surface.elevated}` | oklch(1 0 0) |
| dialog.border | `{alias.border.subtle}` | {map.color.neutral.100} |
| dialog.borderRadius | `{map.radius.lg}` | 27px |
| dialog.padding | `{map.size.lg}` | 20px |

## 5. 사용 케이스 (Do)

- 되돌릴 수 없는 결정 (삭제 확인)
- 사용자 입력이 반드시 필요한 폼

## 6. 사용 금지 (Don't)

- 단순 알림 (`Toast` 사용)
- 폼 길이가 한 화면을 넘어가면 별도 페이지

## 7. 접근성

- `role="dialog"` + `aria-modal="true"` + `aria-labelledby` 필수
- 열린 동안 포커스 트랩 (O-02 위반 회피)
- ESC로 닫기 + 닫힘 후 트리거에 포커스 복귀
- 키보드 트랩 점검 (O-02)
- 다크모드: surface.elevated가 다크에서 충분히 구분되는지 확인

## 8. 코드 예시

```html
<div role="dialog" aria-modal="true" aria-labelledby="m1">
  <h2 id="m1">변경 사항을 저장할까요?</h2>
  <p>토큰 변경 사항이 자동 저장됩니다.</p>
  <button>취소</button>
  <button class="xds-button--primary">확인</button>
</div>
```
