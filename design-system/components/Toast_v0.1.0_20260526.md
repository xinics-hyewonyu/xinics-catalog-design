## 1. 정의

일시적 비차단 알림. 사용자 액션의 결과(성공·실패)나 시스템 이벤트를 화면 모서리에 잠깐 표시 후 자동 사라짐.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `root` | ✓ | 컨테이너 |
| `icon` | – | 톤별 상태 아이콘 |
| `message` | ✓ | 본문 |
| `action` | – | 선택적 액션 버튼 ("실행 취소") |
| `close` | – | 닫기 버튼 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| tone | `info` \| `success` \| `warning` \| `danger` |
| position | `top-right`(기본) \| `bottom-right` \| `bottom-center` |
| duration | 5초(기본) \| `manual`(닫기 전 유지) |

## 5. 사용 케이스 (Do)

- 짧은 결과 알림 ("저장되었습니다")
- 실행 취소 기회 (5초 안에 되돌리기)

## 6. 사용 금지 (Don't)

- 사용자 명시적 결정이 필요한 메시지 → `Modal`
- 긴 본문·복잡한 액션 (가독성 저하)
- 너무 많이 동시에 (3개 이내 권장)

## 7. 접근성

- `role="status"` + `aria-live="polite"`(info/success) 또는 `"assertive"`(danger)
- 자동 사라짐 시간 ≥ 5초 (WCAG 2.2.1)
- 키보드로 닫기 + 액션 가능
- `prefers-reduced-motion`: 슬라이드 인 → 즉시 노출

## 8. 코드 예시

```html
<div role="status" aria-live="polite">
  <span aria-hidden>✓</span>
  저장되었습니다.
  <button>실행 취소</button>
</div>
```

## 9. 관련 컴포넌트

- Alert — 페이지 내 영속 알림 (자동 사라짐 없음)
- Modal — 차단·결정 요구
- Snackbar — 같은 개념의 Material 명칭
