## 1. 정의

트리거(버튼·아바타·아이콘) 옆에 떠올라 *명령·이동 옵션 리스트*를 제공하는 부유 컴포넌트. Dropdown Menu라고도 부른다.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `trigger` | ✓ | 메뉴를 여는 버튼 |
| `list` | ✓ | 옵션 컨테이너 (`role="menu"`) |
| `item` | ✓ | 개별 옵션 (`role="menuitem"`) |
| `divider` | – | 그룹 구분선 |
| `submenu` | – | 중첩 메뉴 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| placement | `bottom-start` \| `bottom-end` \| `top-*` |
| size | `sm` \| `md` |

## 5. 사용 케이스 (Do)

- 사용자 아바타 → 프로필·설정·로그아웃
- 테이블 행의 "⋯" 더보기 액션
- 단계적 명령 그룹 (분류 + divider)

## 6. 사용 금지 (Don't)

- 옵션이 시각적으로 항상 보이는 게 더 명확한 경우 → `Tabs`/`SegmentedControl`
- 옵션 수 15개 초과 (검색·계층화 필요)
- 깊은 다단 중첩 (1단 깊이까지)

## 7. 접근성

- trigger: `aria-haspopup="menu"` + `aria-expanded`
- 컨테이너: `role="menu"`, 항목: `role="menuitem"`
- 키보드: ↑↓ 이동, Enter 실행, Esc 닫기, Tab은 메뉴 닫고 다음 포커스
- 자동 위치 조정 (viewport 벗어남 방지)
- 닫힘 후 트리거에 포커스 복귀

## 8. 코드 예시

```html
<button aria-haspopup="menu" aria-expanded="true">메뉴</button>
<ul role="menu">
  <li role="menuitem">내 프로필</li>
  <li role="menuitem">설정</li>
  <li role="separator"></li>
  <li role="menuitem">로그아웃</li>
</ul>
```
