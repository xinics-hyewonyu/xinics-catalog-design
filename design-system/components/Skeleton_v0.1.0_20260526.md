# Skeleton

> Generated by XDS Generator v0.3.0-rc.1 | 프로젝트: 카탈로그 포트폴리오 | 버전: 0.1.0
> 2026-05-26 — DO NOT EDIT MANUALLY

## 1. 정의

콘텐츠 로딩 중 *실제 레이아웃의 윤곽*을 회색 블록으로 미리 보여주는 placeholder. CLS(레이아웃 시프트) 최소화 + 빠른 응답성 인지.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `block` | ✓ | 사각·원형 placeholder |
| `pulse` | ✓ | 명도 변화 애니메이션 (1.4s loop) |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| shape | `text`(라인) \| `rect` \| `circle` |
| size | `sm` \| `md` \| `lg` |

## 4. 토큰 매핑

| 속성 | 토큰 | 값 |
|---|---|---|
| block.background | `{alias.surface.muted}` | {map.color.neutral.50} |
| pulse.opacity | 0.5~1.0 사이 | – |
| pulse.duration | 1.4s | – |

## 5. 사용 케이스 (Do)

- 카드·리스트·테이블 행 단위 로딩
- 실제 콘텐츠 *너비·높이와 유사*하게 (CLS 방지)

## 6. 사용 금지 (Don't)

- 1초 미만 짧은 로딩 (오히려 산만)
- 페이지 전체를 spinner로 덮어버림 → 부분 Skeleton이 더 친절
- 인터랙티브 컨트롤 위에 (잘못된 클릭 유도)

## 7. 접근성

- 컨테이너에 `role="status"` + `aria-label="콘텐츠 로딩 중"`
- 로딩 완료 시 즉시 실제 콘텐츠로 교체 (DOM 재구성)
- `prefers-reduced-motion`: pulse 정적 색 유지
- 시각 의존 — 스크린리더는 상태 텍스트만 인지

## 8. 코드 예시

```html
<div role="status" aria-label="콘텐츠 로딩 중">
  <span class="xds-skeleton xds-skeleton--text"></span>
  <span class="xds-skeleton xds-skeleton--text"></span>
</div>
```

## 9. 관련 컴포넌트

- Spinner — 단일 인디케이터
- Empty — 로딩 후 데이터 없음 상태
