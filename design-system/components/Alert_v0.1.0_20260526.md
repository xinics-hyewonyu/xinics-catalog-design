## 1. 정의

페이지·섹션 내 *영속적* 알림 배너. Toast(일시 알림)와 달리 자동으로 사라지지 않고 사용자가 명시적으로 닫아야 한다.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `icon` | – | 톤별 상태 아이콘 |
| `title` | ✓ | 상태 요약 |
| `body` | – | 보조 설명 |
| `actions` | – | 인라인 액션 ("자세히" 등) |
| `close` | – | 닫기 버튼 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| tone | `info` \| `success` \| `warning` \| `danger` |
| variant | `subtle`(bg-tinted) \| `outline`(border-only) |

## 5. 사용 케이스 (Do)

- 폼 상단의 검증 오류 요약
- 페이지 진입 시 영구 안내 (서비스 점검 예정 등)
- 컨텍스트 안에 인라인 배치

## 6. 사용 금지 (Don't)

- 일시적 결과 알림 → `Toast`
- 사용자 결정이 필요한 차단 메시지 → `Modal`
- 본문 콘텐츠와 동일 시각 비중 (제목·본문 위계 명확하게)

## 7. 접근성

- tone에 따라 `role`:
  - `info`/`success` → `role="status"` + `aria-live="polite"`
  - `warning`/`danger` → `role="alert"` + `aria-live="assertive"`
- 색만으로 톤 구별 금지 — 아이콘·제목으로 의미 명시 (P-06)
- 텍스트 대비 ≥ 4.5:1 (P-01)
- 닫기 버튼: `aria-label="알림 닫기"`

## 8. 코드 예시

```html
<div role="alert" class="xds-alert xds-alert--warning">
  <span aria-hidden>!</span>
  <div>
    <strong>저장하지 않은 변경이 있습니다</strong>
    <p>페이지를 떠나면 변경 사항이 사라집니다.</p>
  </div>
</div>
```

## 9. 관련 컴포넌트

- Toast — 일시 알림
- Modal — 차단·결정 요구
- Banner — 페이지 최상단 전역 알림
