## 1. 정의

작업의 *최종 상태*(성공·실패·경고)를 페이지 단위로 알리는 큰 인디케이터 + 메시지 + 다음 액션. Empty(데이터 없음)와 달리 *발생한 작업의 결과*를 알린다.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |

| `body` | – | 보조 설명 |
| `actions` | – | 1~2개 다음 액션 |

## 5. 사용 케이스 (Do)

- 결제 완료 화면
- 404·500 오류 페이지
- 회원가입 완료 후 환영 + 다음 액션

## 6. 사용 금지 (Don't)

- 일시적 알림 → `Toast`
- 페이지 내 인라인 알림 → `Alert`

## 7. 접근성

- 아이콘 `aria-hidden`
- 제목 `<h1>` 또는 `<h2>` (페이지 위계와 일관)
- danger 톤은 `role="alert"` + `aria-live="assertive"`
- 다음 액션 CTA는 명확한 동사 ("홈으로" / "다시 시도")
- 색만으로 톤 구별 금지 (P-06)

## 8. 코드 예시

```html
<section class="xds-result xds-result--success">
  <span aria-hidden>✓</span>

  <p>모든 변경 사항이 동기화되었습니다.</p>
  <a class="xds-button--primary" href="/">홈으로</a>
</section>
```

## 9. 관련 컴포넌트

- Alert — 페이지 내 인라인
- Toast — 일시 알림
- Empty — 데이터 없음
