## 1. 정의

아이콘·네비 항목 위에 *오버레이*되는 작은 표시. 카운터(읽지 않은 알림 수) 또는 dot(존재 표시) 두 형태.
Tag와 달리 본문 콘텐츠 안에 들어가지 않고 다른 요소에 *부착*된다.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `host` | ✓ | 부착 대상 (아이콘 등) |
| `indicator` | ✓ | 카운터 숫자 또는 dot |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| kind | `count` \| `dot` |
| tone | `danger`(기본) \| `primary` \| `success` |

## 4. 토큰 매핑 — kind: count, tone: danger

| 속성 | 토큰 | 값 |

| count 9 초과 표시 | "9+", "99+" | – |

## 5. 사용 케이스 (Do)

- 알림 아이콘 위 읽지 않은 메시지 수
- 사이드바 메뉴의 새 항목 표시 (dot)

## 6. 사용 금지 (Don't)

- 본문 콘텐츠 안 분류 라벨 → `Tag`
- 3자리 이상 숫자 노출 ("99+"로 압축)

## 7. 접근성

- `aria-label`로 의미 명시 (예: "읽지 않은 알림 3건")
- 색만으로 의미 전달 금지 (P-06)
- 호스트 요소와 시각 분리(border-ring)

## 8. 코드 예시

```html
<button class="xds-icon-btn" aria-label="알림 3건">
  <svg>...</svg>
  <span class="xds-badge">3</span>
</button>
```

## 9. 관련 컴포넌트
