## 1. 정의

2~4개 옵션을 가로로 압축한 라디오 그룹. 옵션 수가 적고 *항상 보이는 게 더 명확한* 경우 Tabs·Radio·Select를 대체.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `group` | ✓ | `role="radiogroup"` 컨테이너 |
| `segment` | ✓ | 개별 옵션 (`role="radio"`) |
| `indicator` | – | 활성 옵션 배경 강조 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| size | `sm` \| `md` |
| variant | `solid`(활성 배경 highlight) \| `outline` |

## 5. 사용 케이스 (Do)

- 기간 토글 (월/주/일)
- 보기 전환 (그리드/리스트)
- 짧은 옵션 라벨 (1~3자)

## 6. 사용 금지 (Don't)

- 옵션 5개 이상 → `Tabs` 또는 `Select`
- 콘텐츠 전환 의도가 우선 → `Tabs`
- 폼 제출이 필요 → `Radio`

## 7. 접근성

- `role="radiogroup"` + `aria-label` 필수
- 각 `role="radio"` + `aria-checked`
- 키보드: ←→ 로 활성 이동, Tab은 그룹 단위 1회
- 활성 segment는 색뿐 아니라 두께/배경으로 구분 (P-06)

## 8. 코드 예시

```html
<div role="radiogroup" aria-label="기간">
  <button role="radio" aria-checked="false">월간</button>
  <button role="radio" aria-checked="true">주간</button>
  <button role="radio" aria-checked="false">일간</button>
</div>
```

## 9. 관련 컴포넌트

- Tabs — 콘텐츠 영역 전환
- Radio — 폼 제출용 단일 선택
- ToggleGroup — 다중 선택 가능 (체크박스형)
