## 1. 정의

화면 가장자리(우/좌/상/하)에서 슬라이드되어 들어오는 패널. Modal과 달리 본문을 *완전 차단하지 않고 보조 패널*로 작동.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `overlay` | ✓ | 배경 어두운 오버레이 |
| `panel` | ✓ | 슬라이드 패널 |
| `header` | ✓ | 제목 + 닫기 |
| `body` | ✓ | 본문 |
| `footer` | – | 액션 버튼 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| placement | `right`(기본) \| `left` \| `top` \| `bottom` |
| size | `sm`(280) \| `md`(360) \| `lg`(480) |
| dismissible | `true`(기본) \| `false` |

## 5. 사용 케이스 (Do)

- 필터 패널 (테이블·리스트 옆)
- 상세 보기 (선택 항목의 부가 정보)
- 임시 폼 (메인 흐름 차단 없이)

## 6. 사용 금지 (Don't)

- 사용자 결정이 *반드시 필요한* 차단 — `Modal` 사용
- 모바일에서 좌·우 drawer가 메인 콘텐츠 가림 (`bottom` 권장)

## 7. 접근성

- `role="dialog"` + `aria-modal="true"` + `aria-labelledby`
- 포커스 트랩 (Modal과 동일)
- Esc 닫기, 오버레이 클릭 닫기(dismissible)
- 닫힘 후 트리거에 포커스 복귀
- `prefers-reduced-motion`: 슬라이드 → 즉시 표시

## 8. 코드 예시

```html
<div role="dialog" aria-modal="true" aria-labelledby="d-title">
  <header><h2 id="d-title">필터</h2><button>✕</button></header>
  <div>필터 옵션…</div>
</div>
```

## 9. 관련 컴포넌트

- Modal — 명시적 결정 차단
- Popover — 트리거에 부착되는 작은 영역
