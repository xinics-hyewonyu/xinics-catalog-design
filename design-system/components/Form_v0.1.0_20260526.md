# Form

> 2026-05-26 — DO NOT EDIT MANUALLY

## 1. 정의

여러 입력 필드를 묶어 사용자 데이터를 수집하는 컨테이너.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `root` | ✓ | <form> 요소 |
| `field` | ✓ | label + input + helper 묶음 |
| `actions` | ✓ | 제출·취소 버튼 그룹 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| layout | `vertical` \| `horizontal` |
| size | `compact` \| `default` \| `comfortable` |

## 4. 토큰 매핑

Form은 Input·Button·Label 컴포넌트의 조합으로, 별도 페어 토큰을 갖지 않는다.
각 자식 컴포넌트의 토큰 매핑을 참조한다.

## 5. 사용 케이스 (Do)

- 필드 그룹화 (관련 입력 묶음)
- 명시적 제출 액션 (Enter 키 + Submit 버튼 모두 지원)

## 6. 사용 금지 (Don't)

- 포커스 이동만으로 자동 제출 (U-12 위반)
- 라벨 없는 placeholder만 의존 (U-04 위반)

## 7. 접근성

- 모든 필드에 명시적 `<label>` (U-03)
- 오류 메시지는 필드 직후 + `aria-describedby` (U-05)
- 입력 형식 사전 안내 (U-13)
- 제출 실패 시 첫 오류 필드로 포커스 자동 이동
- 한국어 어법·존대 일관 (K-05)

## 8. 코드 예시

```html
<form class="xds-form">
  <label>
    <span>이름</span>
    <input class="xds-input" />
  </label>
  <button class="xds-button--primary">저장</button>
</form>
```

## 9. 관련 컴포넌트

- Input · Select · Textarea · Checkbox · Radio
- FormField — 라벨·입력·헬퍼 묶음
