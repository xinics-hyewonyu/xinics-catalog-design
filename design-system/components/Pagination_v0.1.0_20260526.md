## 1. 정의

다 페이지에 분할된 리스트·표를 이동하는 네비게이션 컨트롤.

## 2. Anatomy

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `prev` | ✓ | 이전 버튼 |
| `pages` | ✓ | 페이지 번호 + 줄임표 |
| `next` | ✓ | 다음 버튼 |
| `summary` | – | "1–20 / 235" 같은 보조 텍스트 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| size | `sm` \| `md` |
| variant | `numbered` \| `simple`(prev·next만) |

## 4. 토큰 매핑 — 현재 페이지

| 속성 | 토큰 | 값 |
|---|---|---|
| current.background | `{alias.action.primary}` | {seed.color.primary} |
| current.color | `{alias.text.onPrimary}` | oklch(1 0 0) |
| page.color | `{alias.text.body}` | {map.color.neutral.900} |
| page.borderRadius | `{map.radius.sm}` | 13px |

## 5. 사용 케이스 (Do)

- 전체 항목 수가 많고 분할이 의미 있는 경우 (검색·로그·관리 화면)
- 페이지 수 ≥ 5 이상부터 노출

## 6. 사용 금지 (Don't)

- 무한 스크롤이 더 적합한 UX (피드·갤러리)
- 페이지 1개만 있을 때 (숨김)

## 7. 접근성

- `<nav aria-label="페이지 네비게이션">` 래퍼
- 현재 페이지: `aria-current="page"`
- 키보드: Tab으로 순회, Enter로 이동

## 8. 코드 예시

```html
<nav aria-label="페이지 네비게이션">
  <button aria-label="이전 페이지">‹</button>
  <button aria-current="page">2</button>
  <button>3</button>
  <button aria-label="다음 페이지">›</button>
</nav>
```

## 9. 관련 컴포넌트

- Table — 페이지네이션이 자주 결합되는 호스트
- LoadMore — 페이지 없이 "더 보기" 버튼
